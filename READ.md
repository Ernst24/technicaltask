## 📂ПРОЕКТ

### 1. Основное приложение

* **Стек:** FastAPI, Pydantic v2, Passlib, Bcrypt, PyJWT
* Серверная часть приложения внутри контейнера.
* Содержит роуты аутентификации (регистрация, логин, логаут, обновление профиля, удаление).
* Содержит кастомный декоратор проверки прав (`PermissionChecker`).
* Содержит Mock-Views для бизнес-объектов (documents, tickets, analytics).

Команда для создания файла конфигурации окружения:
```bash
cp .env.example .env
````
Команда для сборки и запуска всей системы:
```bash
docker-compose up --build
```
Ссылка на интерактивную документацию (Swagger) после запуска:
```bash
🔗 http://127.0.0.1:8000/docs
```
### 2. База данных и Миграции

* Вся история миграций хранится внутри папки `app/migrations` и накатывается автоматически при старте контейнера
* Контейнер автоматически запускает скрипт `app/seed.py`, который наполняет базу ролями, пользователями и 32 правилами доступа

## Тестовые аккаунты (Автоматически создаются в Docker)
Используйте эти данные для проверки ограничений прав в Swagger.
* Общий пароль для всех учетных записей: secret_pass

1. Администратор: admin@mail.ru (Полный доступ ко всему, логам, тикетам и управлению правами)
2. Менеджер: manager@mail.ru (Управление документами, просмотр правил, нет доступа к списку юзеров)
3. Поддержка: support@mail.ru (Доступ к списку пользователей, просмотр и закрытие тикетов)
4. Клиент: client@mail.ru (Базовый юзер, просмотр документов, создание тикетов)
5. Бан: banned@mail.ru (Аккаунт со статусом is_active=False, сразу выдает ошибку 401 Unauthorized)

## Функционал системы

* Динамическая матрица прав в БД (таблицы `roles`, `permissions`, `users`)
* Строгий составной уникальный ключ для правил: `UniqueConstraint(role_id, resource, action)`
* Разделение админских ресурсов: просмотр правил доступа и управление юзерами разнесены на разные сущности

2. Аутентификация и Сессии
* Хеширование паролей через `passlib.context.CryptContext` (схема bcrypt)
* Сессии на JWT-токенах через безопасные HTTP-only Cookies
* Мягкое удаление (Soft Delete) / Бан через флаг `is_active=False`
